# Workflow

## Step 1: Read the project

Start with:

- README
- docs that explain architecture or mechanism
- any public references worth naming in the video

## Step 2: Build the project brief

The brief should contain:

- why
- mechanism
- features
- advantages
- CTA

## Step 3: Create main slides

Main slides carry the narrative skeleton.

## Step 4: Add supporting shots

Supporting shots are optional amplifiers.

## Step 5: Freeze review assets

Before building video:

- approve the slide set
- archive older versions
- keep only the latest review files in the root review folder

## Step 6: Assemble the video

Only after the static assets pass review:

- write narration for each section
- align one spoken section to one visual section
- add limited supporting shots inside that same spoken section
- export subtitles
- render the vertical video
