# Repo Promo Video

This skill helps turn a software project into a short promo-video production pipeline.

## Use this skill when

- the user wants a promo video for a GitHub project
- the user wants to turn README/docs into script + slides + narration
- the user wants reviewable static assets before video assembly

## Workflow

1. Read the repository README and only the docs needed to explain:
   - why it exists
   - how it works
   - what makes it strong
   - what the CTA should be
2. Build a project brief with:
   - why
   - mechanism
   - features
   - advantages
   - CTA
3. Create the main slide set first.
4. Add supporting shots only where the same spoken section needs more emphasis.
5. Get the static slides approved before assembling video.
6. Generate narration, subtitles, cover, and the final vertical video.

## Rules

- Prefer HTML/CSS for dense Chinese slides.
- Do not rely on raw SVG text for long Chinese paragraphs.
- Never overwrite review assets. Version them and move older files into old/.
- Keep supporting shots subordinate to the main slide they serve.
